Description:
------------
Web application that supports creating of adding, editing and deleting automorgues and autoparts. The application supports admin and user accounts.

* jQuery
* KendoUI
* TFS

Project Description
-------------------
Design and implement a SPA application with KendoUI and AngularJS. The application must have a server REST API written in ASP.NET WebAPI or Telerik Everlive.
The application must contain two separate applications – a client application and an admin application.
The client application must be implemented using KendoUI, while the admin application must be implemented with AngularJS.

General Requirements
--------------------
Please define and implement the following assets in your project:

###Requirements for the REST API
* Use ASP.NET WebAPI or Telerik Everlive
  * Expose RESTful endpoints
  * Host the application in AppHarbor, if using WebAPI
* The REST API should expose only CRUD operations over the database and user login/register
 * The business logic of the application should be implemented on the SPA applications
 
###Requirements for the Client SPA application
* Use jQuery (DOM manipulation, AJAX, animations, etc…)
* Implement a MVVM architecture using the framework of KendoUI
* Implement a UI for your application using KendoUI
* The client application should communicate with the services application using HTTP requests
* The application must work in Google Chrome 26,  Mozilla Firefox 21, Internet Explorer 9, Internet Explorer 10, Opera 12
* The client application must have at least 5 pages (routes)
 * Excluding login, register and home pages
 
###Requirements for the Admin SPA application
* Use jQuery (DOM manipulation, AJAX, animations, etc…)
* Implement a MVC architecture using AngularJS
* The application should communicate with the services application using HTTP requests
* The application must work in Google Chrome 26,  Mozilla Firefox 21, Internet Explorer 9, Internet Explorer 10, Opera 12
* The admin application must have at least 2 pages (routes)
 * Excluding login and register pages

Additional Requirements
-----------------------
* Follow the best practices for OO design: use data encapsulation, use exception handling properly, use inheritance, abstraction and polymorphism properly and follow the principles of strong cohesion and loose coupling.
